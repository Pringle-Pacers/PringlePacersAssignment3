Testing

Overview

The project will be tested be via user acceptance testing which will accomplished by providing a prototype of the application towards end users, as this will be the most time efficient and cost-effective method to determine if we have accomplished the scope of the application as well as to iron out and errors, or to alter the program and add more features. The first stage of the UAT testing is gathering the appropriate users, which will be done anonymously to prevent information bias which prevents an accurate representation of feedback among the participants. The second stage involves gathering the relevant feedback from the users. The specific issues we may be looking for are validation and functional problems with the program, and potential improvement to the ergonomics of the application. We will have succeeded in testing, when we have gathered data from the individual participants that is valuable enough to be synthesised into information that will provide an accurate perspective on the positive and negative aspects of the application and whether the application has performed as what the user has expected, and to what we as the developers have intended. This is acceptable once the feedback of the participants has been reciprocated such as having pushed a feature such as achievements to a greater significance if it is popular or high demand among the participants.

Test 1

TEST 1: Have the participant sign up with a predetermined set of details and click sign up.
Completion Condition: The predetermined set of details have been saved into the system and should now allow the user to login with these set of details. These details should also display in the profile window.

Test 2

TEST 2: Have the participant login with the same set of predetermined details from TEST 1 and click login.
Completion Condition: The participant has successfully logged in with the predetermined details and access windows otherwise not viewable when the participant is not logged in which is the Select your workout window, the exercise randomiser window, the exercise list page, the achievements page and the profile/edit window.

Test 3

TEST 3: Have the participant edit the pre-existing details of the profile in the profile edit window with another set of predetermined details and have the participant save these details.
Completion Condition: The participant can edit the text fields and has successfully updated the details of the profile with the new set of details and these new details now show on the profile window.

Test 4

TEST 4: Within the BMI Calculator window, have the participant select the male for between the two options, then add a height (186cm) and a weight (80kg) in the respective text fields.
Completion Condition: The user is successfully able to enter the predetermined inputs, in the selection and text fields and the section to the right of the input fields should show the associated BMI calculated from the numbers and qualities put in the input fields which should come to 23.1. This should also be accompanied by text informing the participant of what BMI range they are in (normal) and displaying an image that is associated with that weight range.

Test 5

TEST 5: Within the workout selection window, have the participant select a chest workout from pre-set workouts.
Completion Condition: The exercises associated with the chest workout should now show in the My List page.

Test 6

TEST 6: Within the My list window, have the participant select the complete button after having checked off every module in the workout list.
Completion Condition: The My List window should clear the existing workout modules and then register these completed workouts into the achievements section, displaying the progression increase in each section respective to the exercises completed, determined on the workout list that had been completed.
